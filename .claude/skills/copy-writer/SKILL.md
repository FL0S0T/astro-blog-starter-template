# Copy Philosophy

## Authority Over Persuasion

This is not aggressive marketing copy.

Goal:

- Positioning clarity
- Trust
- Intellectual authority
- Strategic relevance

Not:

- Forced conversion
- Emotional persuasion
- Artificial urgency

---

## Clarity Over Cleverness

Avoid:

- Wordplay
- Slogans
- Marketing phrasing

Prefer:

- Clear identity statements
- Structural arguments
- Direct positioning

---

## Personal but Precise

Allowed:

- "Ich baue…"
- "Ich code gerade…"
- "In meiner Beratungszeit habe ich gelernt…"
- "Als Vater von zwei Kindern…"

But keep it concise.
Return to structural substance quickly.

# Page-Specific Guidelines

Purpose:
Immediate clarity.

Structure:

- Strong identity headline
- Clear positioning paragraph
- Featured articles with hero images
- Five thinking layers
- Soft but confident CTA

Headline examples:

- I build companies, write code and think structurally.
- From consulting to code. From M&A to AI.

Explain what readers gain — not what you feel.

CTA examples:

- Read articles
- Operator Letter
- Artikel lesen
- Operator Letter abonnieren

## About Page

Purpose:
Trust and credibility.

Include:

- Profile photo
- Who you are (hands-on CEO, tech enthusiast, builder)
- Background (Ausbildung, TU Darmstadt WINF, consulting)
- What you do (coding, cloud, AI, building companies)
- Why you write
- What drives you

- Chronological CV narrative
- Emotional founder story
- Self-praise

# CTA Philosophy

Avoid:

- Submit
- Get Started
- Secure your spot
- Limited offer

Prefer:

- Artikel lesen / Read articles
- Operator Letter abonnieren / Subscribe
- Gespräch anfragen / Start a conversation
- Gedanken vertiefen

CTAs should feel calm and confident.

---

# Headline Rules

Headlines must:

- Clarify identity
- Signal competence
- Avoid exaggeration
- Avoid emotional framing

Bad:
"Building the future of AI."

Good:
"I build companies, write code and think structurally."
"From consulting to code — structural thinking in the AI age."
"Technologie strukturell denken."

---

# Credibility Strategy

Instead of loud proof:

Avoid:
"Trusted by leading companies."

Prefer:
"I write from the perspective of a hands-on builder — not from theory."
"From M&A to AI implementation — I build what I write about."

Authority through substance and demonstrated experience.

---

# Anti-Patterns

Never:

- Sound like a SaaS landing page.
- Use startup buzzwords.
- Add fake scarcity.
- Use emotional hooks.
- Overexplain basics.

If it sounds like marketing — simplify.
If it sounds corporate — humanize.
If it sounds vague — make it precise.

---

# Final Standard

All copy must reflect:

- Executive clarity
- Builder mindset
- Hands-on technical authenticity
- Structural thinking
- Personal grounding
- Calm authority
- Analytical depth

If it feels soft — sharpen.
If it feels loud — reduce.
If it feels vague — structure.
